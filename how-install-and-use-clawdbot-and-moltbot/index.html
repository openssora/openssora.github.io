教你如何安装和使用 Clawdbot 和 Moltbot 的保姆级教程（2026年最新）

最近几天相信大家都被 Clawdbot 疯狂的刷屏了，跟去年的 Manus 发布时差不多一样，在互联网上爆发式的展开。

原先是小龙虾，现在的 Logo 是脱了壳的小龙虾，已命名为：Moltbot。

这是因为 Clawd 的读音跟 Claude 商标冲突，被 Anthropic 给告了，被迫改为 Moltbot 。

Anthropic 认为 Clawdbot 这个名字太容易被市场误解为 Claude Code的衍生产品，所以 Clawdbot 已改名为 Moltbot。

Moltbot 取自龙虾的脱壳行为，符合原本的龙虾元素，并且规避了 Claude 的商标问题。

什么是 Clawdbot 呢？

Clawdbot 是一个 AI 助手，可以部署在本地电脑上或者服务器，可以理解为是一个拥有系统超级权限（root）的 Agent。

如果你用过 Claude Code， 就可以把它理解为拥有各种 skills 以及可以调用各种 MCP 服务协助完成任务的闭环。

可以让它帮你整理邮件、管理日程、炒股、写推文、读PPT等，在后台24小时运行，但这是非常耗 Token。

因为 Clawdbot 拥有的权限比较高，一不小心可能会把电脑系统的东西删除了就麻烦啦！！！

另外 Clawdbot 的安全性堪忧，直接部署和一些线上服务联合在一起，你的 Clawdbot 大部分网关可是暴露在公网上的，没有任何身份验证，就可以拥有完整的 Shell 访问权限。 小心黑客直接偷家了哦

需要尝试的伙伴，建议在虚拟机上部署，或者在一台不怎么用的 Mac 电脑也行。

最后，问你需不需要开启钩子，这三个都是 Moltbot 的内置 hooks（钩子脚本），用来在特定事件发生时自动做事：

  • boot-md：网关（gateway）启动后自动读取并执行工作区里的 BOOT.md 里的引导/启动指令。
  • command-logger：把所有命令事件记录到本地审计日志文件（默认 ~/.clawdbot/logs/commands.log）
  • session-memory：当你发出 /new 时，把当前会话上下文保存成一份“记忆快照”到工作区（默认 ~/clawd/memory/ 相关路径）。
